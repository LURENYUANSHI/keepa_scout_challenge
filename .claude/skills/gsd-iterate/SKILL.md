# GSD Iterate — 自动迭代修复循环

## 用途

在 AI 完成代码编写或修改后，自动执行以下循环直到代码质量达标：

```
写代码/修复 → 运行测试 → 代码评审 → 风险 Low? → 提交
                ↑                          ↓ No (有问题)
                └──────── 自动修复 ←───────┘
                         (最多 N 次，默认 5)
```

## 调用方式

```
/gsd-iterate                    # 对当前未提交变更执行迭代
/gsd-iterate --max=3            # 最多 3 轮迭代
/gsd-iterate "修复 parser.py"   # 先执行任务描述，再进入迭代循环
```

## 工作流（严格按顺序）

| 步骤 | 说明 |
|------|------|
| 0 | 解析参数，确定 max_iterations（默认 5） |
| 1 | 收集当前变更（若有任务描述先执行任务） |
| 2 | **迭代循环** |
| 3 | 最终提交或上报 |

---

## 详细步骤

### Step 0 — 解析参数

从 `$ARGUMENTS` 解析：
- `--max=N` → `MAX_ITER=N`（默认 5，范围 1-10）
- 其余文本 → `TASK_DESC`（任务描述，可为空）

显示 banner：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► ITERATE  (max {MAX_ITER} rounds)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 1 — 准备变更

**若 `TASK_DESC` 非空：**
先按描述完成代码修改（Edit/Write 相关文件），然后继续。

**若 `TASK_DESC` 为空：**
直接使用当前工作区已有的未提交变更（`git diff HEAD` 非空即可继续）。

若工作区无任何变更且无任务描述 → 提示用户并退出。

### Step 2 — 迭代循环

初始化：`ITER=1`，`STATUS=pending`

**每轮循环执行：**

#### 2a. 运行测试（auto-test-writer）

按 auto-test-writer skill 完整执行：
1. 收集变更函数清单
2. 为每个新增/修改函数生成测试（正常路径 + 边界值 + 异常输入）
3. 放置测试文件（追加到已有测试文件）
4. 运行全量测试套件：`pytest -q --tb=short`
5. 记录结果：`TEST_RESULT=pass|fail`，`FAILED_TESTS=[]`

**若测试失败（TEST_RESULT=fail）：**
- 记录失败的测试和错误信息到 `ISSUES`
- 跳过 2b，直接进入 2c 修复（测试失败 > 代码有 bug）

#### 2b. 代码评审（code-review-eval）

仅在测试全部通过后执行：
1. 收集本次变更文件
2. 执行 5 项检查：最小化 / 副作用 / 代码质量 / 破坏性变更 / 测试覆盖
3. 运行 lint（ruff）和类型检查（mypy）
4. 汇总风险等级：`RISK=Low|Medium|High|Critical`
5. 记录所有 findings 到 `ISSUES`

**若 `RISK=Low` 且 `TEST_RESULT=pass`：**
- `STATUS=done`，跳出循环 → 进入 Step 3

#### 2c. 自动修复

**触发条件：** `TEST_RESULT=fail` 或 `RISK >= Medium`

按 `ISSUES` 中的每项 finding 执行修复：

| 问题类型 | 修复动作 |
|----------|----------|
| 测试失败 | 定位失败行，修复源码 bug（优先修源码而非改测试） |
| ruff lint 报错 | 按错误提示直接 Edit 源码修复 |
| mypy 类型错误（本次引入的） | 修正类型注解或实现 |
| 最小化违规 | 回退多余改动 |
| 副作用/破坏性变更 | 恢复签名兼容性或加默认值 |
| 覆盖率不足 | 补充测试用例 |
| 代码质量（hardcode/命名/重复） | 提取常量/改名/抽函数 |

**修复优先级：** 测试失败 > Critical > High > Medium > Low

修复完毕后，打印当前轮次总结：
```
[Iter {ITER}/{MAX_ITER}] Tests: {PASS|FAIL} | Risk: {LEVEL} | Fixed: {N} issues
```

#### 2d. 检查是否继续

```
ITER += 1
if ITER > MAX_ITER:
    STATUS=max_reached
    跳出循环
else:
    回到 2a
```

### Step 3 — 最终处理

**STATUS=done（全部通过）：**

执行提交：
```bash
git add <changed_files>
git commit -m "fix/feat: <auto-generated message from changes>"
```

输出完成报告：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► ITERATE COMPLETE ✓
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Rounds:   {ITER-1} / {MAX_ITER}
Tests:    PASS ({N} tests)
Risk:     Low
Commit:   {hash}
```

**STATUS=max_reached（超过最大轮数）：**

不提交。上报给用户：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► ITERATE STALLED ({MAX_ITER} rounds exhausted)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Remaining issues (need human judgment):
{ISSUES list}

Options:
1. /gsd-iterate --max=3   (继续迭代)
2. Review manually        (人工检查上面的问题)
3. git stash              (放弃本次变更)
```

---

## 约束

- **永远不跳过测试** — 即使评审 Low，测试未全过不得提交
- **优先修源码而非改测试** — 测试失败时不得删除或弱化断言
- **每轮修复必须针对 ISSUES 中的具体条目** — 禁止盲目重写
- **最多 10 轮**（`--max` 上限），防止无限循环
- **mypy 预存在错误不计入 ISSUES** — 仅本次引入的新错误需修复

## 与其他 skill 的关系

| Skill | 关系 |
|-------|------|
| `auto-test-writer` | Step 2a 内嵌执行其完整流程 |
| `code-review-eval` | Step 2b 内嵌执行其完整流程 |
| pre-commit gate | `gsd-iterate` 完成后提交可绕过 gate（已内置了相同检查） |
