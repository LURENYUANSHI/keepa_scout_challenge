# 禁止事项

1. **禁止跳过 review 直接 commit** —— 即使用户催促
2. **禁止跳过 auto-test-writer** —— 本 skill 的前置条件是测试已全绿
3. **禁止只评审未通过的子项** —— 修复后必须从 Step 1 重跑全部检查
4. **禁止通过降低评估标准让风险"看起来"变 Low** —— 只能通过修代码降风险
5. **禁止评审未触及的历史代码** —— 质量检查只看本次 diff 行
6. **禁止 commit 任何疑似密钥 / token / 密码的硬编码字符串** —— 直接 Critical
7. **禁止在发现 breaking change 后不加兼容层直接让它过** —— 除非用户明确要求 breaking
8. **禁止吞异常** —— 发现 `except: pass` / `catch {}` 必须修掉
9. **禁止无限重跑** —— 连续 3 轮仍未降到 Low，停下来向用户汇报
10. **禁止顺手"优化"review 中发现的无关问题** —— 那是下一个任务的事，本次只修本次变更
