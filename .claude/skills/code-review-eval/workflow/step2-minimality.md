# Step 2 - 最小化修改检查

## 核心问题

**每一处改动，是否都是完成"本次任务意图"所必需的？**

## 逐处审视 diff 的 hunk

对 Step 1 清单中的每个 hunk，问：

1. 这行是否直接服务于本次任务？
2. 能否删掉它而任务依然完成？若能，它就是"多余改动"
3. 它是不是顺手做的无关优化 / 重构 / 格式化 / 注释更新？

## 必须回退的改动类别

- 顺手重命名无关的变量 / 文件
- 顺手重构无关的函数结构
- 顺手添加 / 修改无关的注释或 docstring
- 顺手格式化未改动的行（纯空白 / 引号 / 尾逗号 diff）
- 给"没改动的代码"补类型注解
- 引入的新依赖实际没用到
- 留下的调试代码：`print` / `console.log` / `TODO` / 注释掉的代码块
- 顺手"修复"了另一个不相关的 bug（应走单独 commit）

## 允许的非任务改动（白名单）

- 为新增函数添加必要的 import
- 为修改的函数补充必要的类型注解/docstring（若项目约定要求）
- 修复因本次改动而引入的 lint 错误

## 操作

发现多余改动 → **立即回退**（用 Edit 恢复原样；从 Read 过的原文件取基准，或用 `git diff` 对照）。
