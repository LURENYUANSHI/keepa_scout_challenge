# Step 6 - 提交前自检

在执行 `git commit` **之前**，逐项确认：

- [ ] step1 变更清单中的**每一个**新增/修改函数都已覆盖？
- [ ] 每个函数都有"正常路径 + 边界值 + 异常输入"各至少 1 个 case？
- [ ] 新测试文件遵循了项目原有目录结构和命名约定？
- [ ] 增量测试通过？
- [ ] 全量测试通过（没有回归）？
- [ ] 没有使用 skip / xfail / 注释掉断言 来绕过失败？
- [ ] 没有引入未经用户同意的新依赖？
- [ ] 新生成的测试文件已 `git add`，会一并进入本次提交？

**任一项未通过 → 返回对应步骤修复，禁止提交**。

全部通过后，向用户用一句话汇报："已为 N 个函数生成 M 个测试用例，全部通过"，然后继续原本的 commit 流程。
