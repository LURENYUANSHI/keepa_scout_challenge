# 禁止事项

1. **禁止跳步** —— 工作流 6 个步骤必须顺序完成，测试未全绿不得进入下一步
2. **禁止绕过失败** —— 不得使用 `skip` / `xfail` / 注释断言 / `try-except: pass` / 放宽期望值等方式"让测试通过"
3. **禁止无意义断言** —— 禁止 `assert result is not None`、`assert True`、只调用函数不断言等套路
4. **禁止过度 mock** —— 能用真实简单数据就不要 mock；只对真正的外部副作用（网络、DB、FS、时间、随机）做 mock
5. **禁止擅自新增依赖** —— 不得在未经用户同意的情况下修改 `package.json` / `pyproject.toml` / `go.mod` 新增测试依赖
6. **禁止删改无关测试** —— 只追加或修改与本次变更相关的测试，不要顺手"优化"其他测试
7. **禁止擅自重构源码** —— 修 bug 时改动范围最小化，不得借机重构
8. **禁止盲目循环修改** —— 连续 2 次修改仍未通过时必须停下来分析或求助，不要无脑重试
9. **禁止在测试未通过时 commit** —— 这是本 skill 存在的根本原因
10. **禁止编造项目约定** —— 若不确定测试目录/命名/运行命令，先 Read / Glob 现有测试确认
